# Test Cases Directory

This directory stores test cases for each GitHub issue.

## Naming Convention
- File format: `issue-{number}.md`
- Example: `issue-5.md` for issue #5

## When Test Cases Are Created
Test cases are created by the QA agent immediately after the Product Manager creates a new issue.

## When Test Cases Are Used
Test cases are referenced by the QA agent when testing features in the "QA Testing" status.

## When Test Cases Are Updated
Test cases can be updated by QA agent when:
- New edge cases are discovered during testing
- Acceptance criteria change
- Additional scenarios need to be tested
- Previous test cases need clarification

## File Structure
Each test case file includes:
- Issue reference and feature description
- Test scenarios (TC-1, TC-2, etc.)
- Edge cases checklist
- Regression tests checklist
- Test execution history (Pass/Fail status and dates)
